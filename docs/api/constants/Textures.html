Texture Constants

Mapping Modes（映射模式）

THREE.UVMapping THREE.CubeReflectionMapping THREE.CubeRefractionMapping THREE.EquirectangularReflectionMapping THREE.EquirectangularRefractionMapping THREE.SphericalReflectionMapping THREE.CubeUVReflectionMapping THREE.CubeUVRefractionMapping
这些定义了纹理的映射模式。
[page:Constant UVMapping] 是默认值, 使用网格的 UV 坐标映射纹理。

其余的定义了环境映射类型。

[page:Constant CubeReflectionMapping] 和 [page:Constant CubeRefractionMapping] 用于 [page:CubeTexture CubeTexture], 一个 CubeTextture 由六个纹理组成，每一个对应于立方体的一面。
[page:Constant CubeReflectionMapping] 是 [page:CubeTexture CubeTexture] 的默认值。

[page:Constant EquirectangularReflectionMapping] 和 [page:Constant EquirectangularRefractionMapping] 用于等矩形环境映射。 也被称为经纬映射， 一个等矩形纹理表示一个沿水平中间线的 360 度视图，和一个沿垂直轴的 180 度视图， 图像的顶部和底部边缘对应于映射球体的南北极。

[page:Constant SphericalReflectionMapping] 用于球面反射映射， 如剪切镜像球的照片即可获得球面映射会被渲染成正面相机， 无关相机相对于立体映射的物体或表面的位置。

参见示例 [example:webgl_materials_envmaps materials / envmaps]。

Wrapping Modes（循环模式）

THREE.RepeatWrapping THREE.ClampToEdgeWrapping THREE.MirroredRepeatWrapping
这些定义了纹理的 [page:Texture.wrapS wrapS] 和 [page:Texture.wrapT wrapT] 属性， 两者定义了水平和垂直方向上的纹理循环模式。

使用 [page:constant RepeatWrapping]，纹理会简单地重复无穷次。

[page:constant ClampToEdgeWrapping] 是默认值。 纹理的最后一个像素延伸到网格的边缘。

使用 [page:constant MirroredRepeatWrapping]，纹理会重复无穷次，每次重复都做一次镜像翻转。

Magnification Filters（放大滤镜）

THREE.NearestFilter THREE.LinearFilter
用于纹理的 [page:Texture.magFilter magFilter] 属性, 当被纹理化的像素映射到小于或者等于一个纹理元素的区域（texel）时, 这些定义了要使用的纹理的放大函数。

[page:constant NearestFilter] 返回一个距离指定纹理坐标最近 （曼哈顿距离）的纹理元素值。

[page:constant LinearFilter] 是缺省值， 返回最接近指定纹理坐标的四个纹理元素的加权平均值， 包括一个纹理的其他部分覆盖或重复的项， 具体取决于 [page:Texture.wrapS wrapS] 和 [page:Texture.wrapT wrapT] 的值, 以及精确的映射。

Minification Filters（缩小过滤器）

THREE.NearestFilter THREE.NearestMipMapNearestFilter THREE.NearestMipMapLinearFilter THREE.LinearFilter THREE.LinearMipMapNearestFilter THREE.LinearMipMapLinearFilter
用于纹理的 [page:Texture.minFilter minFilter] 属性，这些属性定义了纹理缩小函数， 每当纹理像素映射到大于一个纹理元素（texel）的区域时使用这些函数。

除了 [page:constant NearestFilter] 和 [page:constant LinearFilter]， 还可以使用以下四个函数进行缩小：

[page:constant NearestMipMapNearestFilter] 选择与纹理像素大小最匹配的 mipmap， 并使用 [page:constant NearestFilter] 准则（最接近像素中心的 texel） 生成纹理值。

[page:constant NearestMipMapLinearFilter] 选择两个与纹理像素大小最匹配的 mipmaps 并使用 [page:constant NearestFilter] 准则从每个 mipmap 生成纹理值。 最终纹理值是这两个值的加权平均值。

[page:constant LinearMipMapNearestFilter] 选择与纹理像素大小最匹配的 mipmap， 并使用 [page:constant LinearFilter] 准则 （最接近像素中心的四个 texel 的加权平均值） 生成纹理值。

[page:constant LinearMipMapLinearFilter] 是默认设置，它选择与纹理像素大小最匹配的两个 mipmap， 并使用 [page:constant LinearFilter] 准则 从每个 mipmap 生成纹理值。最终纹理值是这两个值的加权平均值

示例参见 [example:webgl_materials_texture_filters materials / texture / filters]。

Types（类型）

THREE.UnsignedByteType THREE.ByteType THREE.ShortType THREE.UnsignedShortType THREE.IntType THREE.UnsignedIntType THREE.FloatType THREE.HalfFloatType THREE.UnsignedShort4444Type THREE.UnsignedShort5551Type THREE.UnsignedShort565Type THREE.UnsignedInt248Type
用于纹理的 [page:Texture.type type] 属性，该属性必须和正确的格式相对应。请参阅下文了解详情。

[page:constant UnsignedByteType] 是默认值。

Formats（格式）

THREE.AlphaFormat THREE.RGBFormat THREE.RGBAFormat THREE.LuminanceFormat THREE.LuminanceAlphaFormat THREE.RGBEFormat THREE.DepthFormat THREE.DepthStencilFormat
用于纹理的 [page:Texture.format format] 属性，这些属性定义了 着色器如何读取 2d 纹理或者 *texels* 的元素。

[page:constant AlphaFormat] 丢弃红色，绿色和蓝色分量而只读取 alpha 分量。 [page:Texture.type type] 必须是 [page:constant UnsignedByteType]。

[page:constant RGBFormat] 丢弃 alpha 分量而读取红色，绿色和蓝色分量。 [page:Texture.type type] 必须是 [page:constant UnsignedByteType] 或者 [page:constant UnsignedShort565Type]。

[page:constant RGBAFormat] 是默认值，可读取红色，绿色，蓝色和 alpha 分量。 [page:Texture.type type] 必须是 [page:constant UnsignedByteType]，[page:constant UnsignedShort4444Type] 或者 [page:constant THREE.UnsignedShort5551Type].

[page:constant LuminanceFormat] 将每个元素读取为亮度分量。reads each element as a single luminance component. 之后将其转换为浮点型，限制在 [0,1] 范围内，, and then assembled 然后通过将亮度值放在红色，绿色和蓝色通道中并将 1.0 附加到 alpha 通道中，将其组装成 RGBA 元素。 [page:Texture.type type] 必须是 [page:constant UnsignedByteType].

[page:constant LuminanceAlphaFormat] 将每个元素读取为一个表示亮度/alpha 的浮点型值。 除了 alpha 可能具有 1.0 以外的值，与 [page:constant LuminanceFormat] 具有相同的处理过程。 [page:Texture.type type] 必须是 [page:constant UnsignedByteType].

[page:constant RGBEFormat] 与 [page:constant RGBAFormat] 相同。

[page:constant DepthFormat] 将每个元素作为单个深度值读取，转换成浮点型，并限制在 [0,1] 范围内。 [page:Texture.type type] 必须是 [page:constant UnsignedIntType] 或者 [page:constant UnsignedShortType]。 这是 [page:DepthTexture DepthTexture] 的默认值。

[page:constant DepthStencilFormat] 将每个元素作为一对深度和模板值读取。 这对值的深度分量在 [page:constant DepthFormat] 有所解释。 模板分量是在深度 + 模板的内部格式基础上被解释的。 [page:Texture.type type] 必须是 [page:constant UnsignedInt248Type].

请注意纹理必须有如上所述的正确 [page:Texture.type type] 集合。 请参阅 此页面 获取更多详情。

DDS / ST3C Compressed Texture Formats（DDS / ST3C 压缩纹理格式）

THREE.RGB_S3TC_DXT1_Format THREE.RGBA_S3TC_DXT1_Format THREE.RGBA_S3TC_DXT3_Format THREE.RGBA_S3TC_DXT5_Format
用于 [page:CompressedTexture CompressedTexture] 的 [page:Texture.format format] 属性， 这些属性需要支持 WEBGL_compressed_texture_s3tc 扩展。
据 WebglStats 统计，截至 2016 年 2月， 超过 80% 的支持 WebGL 的设备支持此扩展。

通过此扩展可以使用四种 S3TC 格式。它们是：
[page:constant RGB_S3TC_DXT1_Format]: RGB 图像格式的 DXT1 压缩图像。
[page:constant RGBA_S3TC_DXT1_Format]: RGB 图像格式的 DXT1 压缩图像，具有简单的 on/off alpha 值。
[page:constant RGBA_S3TC_DXT3_Format]: RGBA 图像格式的 DXT3 压缩图像。与 32 位的 RGBA 纹理相比，它提供 4:1 的压缩。
[page:constant RGBA_S3TC_DXT5_Format]: RGBA 图像格式的 DXT5 压缩图像。它还提供 4:1 压缩，但与 DXT3 压缩的不同之处在于如何进行 alpha 压缩。

PVRTC Compressed Texture Formats（PVRTC 压缩纹理格式）

THREE.RGB_PVRTC_4BPPV1_Format THREE.RGB_PVRTC_2BPPV1_Format THREE.RGBA_PVRTC_4BPPV1_Format THREE.RGBA_PVRTC_2BPPV1_Format
用于 [page:CompressedTexture CompressedTexture] 的 [page:Texture.format format] 属性， 这些属性需要支持 WEBGL_compressed_texture_pvrtc 扩展。
据 WebglStats 统计，截至 2016 年 2 月， 只有不到 8% 的支持 WebGL 的设备支持这种扩展。 PVRTC 通常仅适用于具有 PowerVR 芯片组的移动设备， PowerVR 芯片组主要 Apple 设备。

通过此扩展可以使用四种 PVRTC 格式。它们是：
[page:constant RGB_PVRTC_4BPPV1_Format]: 4 位模式下的 RGB 压缩。每个 4×4 像素一个块。
[page:constant RGB_PVRTC_2BPPV1_Format]: 2 位模式下的 RGB 压缩。每个 8×4 像素一个块。
[page:constant RGBA_PVRTC_4BPPV1_Format]: 4 位模式下的 RGBA 压缩。每个 4×4 像素一个块。
[page:constant RGBA_PVRTC_2BPPV1_Format]: 2 位模式下的 RGBA 压缩。每个 8×4 像素一个块。

ETC Compressed Texture Format（ETC 压缩纹理格式）

THREE.RGB_ETC1_Format
用于 [page:CompressedTexture CompressedTexture] 的 [page:Texture.format format] 属性， 这些属性需要支持 WEBGL_compressed_texture_etc1 扩展。
据 WebglStats 统计，截至 2016 年 2 月， 只有超过 13% 的支持 WebGL 的设备支持这种扩展。

Encoding（编码）

THREE.LinearEncoding THREE.sRGBEncoding THREE.GammaEncoding THREE.RGBEEncoding THREE.LogLuvEncoding THREE.RGBM7Encoding THREE.RGBM16Encoding THREE.RGBDEncoding THREE.BasicDepthPacking THREE.RGBADepthPacking
用于纹理的 [page:Texture.encoding encoding] 属性.

如果在材质已使用纹理后更改编码类型， 你需要将 [page:Material.needsUpdate Material.needsUpdate] 设置为 *true* 以使材质重新编译。

[page:constant LinearEncoding] 是默认值， 除此之外的值仅对材质的 map，envMap 和 emissiveMap 有效。
