从外部文件加载的内容

如果你从外部文件加载模型或纹理，由于浏览器的[link:http://en.wikipedia.org/wiki/Same_origin_policy 同源策略] 安全限制，从文件系统加载会失败，并出现安全异常。

有两种方法可以解决这个问题：

  1. 修改浏览器对本地文件的安全策略。这可以让你访问你的页面：file:///yourFile.html
  2. 从本地 web 服务器运行文件。这可以让你访问你的页面：http://localhost/yourFile.html

如果你使用选项1，请注意，如果使用 相同的浏览器进行常规 web 访问，你可能面对一些安全漏洞。你不妨创建一个单独的浏览器 配置 / 快捷方式仅用于本地开发以确保安全。我们依次浏览每个选项。

运行本地服务器

许多编程语言都内置了简易的 HTTP 服务器。他们不像 [link:https://www.apache.org/ Apache] 或 [link:https://nginx.org NGINX] 那样功能全面，但他们应该 足以用来测试你的 three.js 应用。

Node.js 服务器

Node.js 有一个简单的 HTTP server 包。安装：

npm install http-server -g

开始运行（从你的本地目录）：

http-server . -p 8000

Python 服务器

如果你安装了 [link:http://python.org/ Python]， 通过命令行（从你的工作目录）运行它应该足够了：

//Python 2.x python -m SimpleHTTPServer //Python 3.x python -m http.server

这样可以通过本地主机的8000端口访问当前目录的文件，即在地址栏中输入：

http://localhost:8000/

Ruby 服务器

如果你安装了 Ruby，你可以运行以下内容获得相同结果：

ruby -r webrick -e "s = WEBrick::HTTPServer.new(:Port => 8000, :DocumentRoot => Dir.pwd); trap('INT') { s.shutdown }; s.start"

PHP 服务器

PHP 从 php 5.4.0 起也有内置服务器：

php -S localhost:8000

Lighttpd

Lighttpd是一个非常轻量级的通用网络服务器。 我们将在这里介绍在 OSX 上使用 HomeBrew 安装它。 与此处讨论的其他服务器不同，lighttpd 是一个成熟的可用于生产 环境的服务器。

  1. 通过 homebrew 安装它 brew install lighttpd
  2. 在要运行Web服务器的目录中创建一个名为 lighttpd.conf的配置 文件。这里有个[link:http://redmine.lighttpd.net/projects/lighttpd/wiki/TutorialConfiguration 例子]。
  3. 在 conf 文件中，将 server.document-root 更改为你要为文件提供服务的目录。
  4. 启动它 lighttpd -f lighttpd.conf
  5. 导航到 http://localhost:3000/ ，它将为你选择的目录中的静态文件提供 服务。

更改本地文件安全策略

Safari

使用首选项面板启用 Develop 菜单，在高级 -> “在菜单栏中 显示 Develop 菜单”。

然后在 safari "Develop" 菜单中，选择“禁用本地文件限制”， 同样值得注意的是 Safari 浏览器对缓存有怪异表现，所以如果你正在使用 safari 编辑&调试， 建议在同一菜单中启用“禁用缓存”选项。

Chrome

首先关闭所有运行中的 Chrome 实例，重点是“所有”。

在 Windows 中，你可以使用 Windows 任务管理器检查 Chrome 实例。 此外，如果你在系统托盘中看到 Chrome 图标，那么你可以打开它的右键 菜单并点击“退出”。这应该可以关闭所有 Chrome 实例。

然后使用命令行标志启动 Chrome 可执行文件：

chrome --allow-file-access-from-files

在 Windows 上，最简单的方式可能是创建一个添加了上述命令行标志 的特殊快捷方式（右键单击快捷方式 -> 属性 -> 目标）。

在 Mac OSX 上，你可以使用

open /Applications/Google\ Chrome.app --args --allow-file-access-from-files

Firefox

  1. 在地址栏中，输入 about:config
  2. 找到 security.fileuri.strict_origin_policy parameter
  3. 将其设置为 false

在 Stack Overflow 上有其他简单的选项 的[link:http://stackoverflow.com/q/12905426/24874 讨论]。
